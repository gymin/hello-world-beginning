Ford GoBike Data Exploration¶

by Geum Young Min¶

Preliminary Wrangling¶

This document explores the Ford GoBike dataset. Ford GoBike is a bike sharing system in the Bay Area, USA. People can become an annual member or buy a single ride or access pass to use public bikes available at any Ford GoBike station. With an annual membership a member (or a 'subscriber') can use a bike for unlimited 45 minute-trips, while a causal user (a.k.a 'customer') can enjoy one ride up to 30 minutes ('Single Ride') or unlimited 30-minute-trip in a 24 hour period ('Access Pass'). The following documents contain Ford GoBike's trip data until April 2019.

What is the structure of your dataset?¶

There are 239,111 trip data with 16 features (trip duration in sec, start time and date, end time and date, start station ID, start station name, start station latitude, start station longitude, end station id, end station name, end station latitude, end station longitude, bike ID, user type, member year of birth, member gender, bike share for all trip). Most variables are categorical in nature, and although some variables numeric, probably one most meaningul numeric variable would be trip duraiton.

What is/are the main feature(s) of interest in your dataset?¶

I am interested in finding out who are the main users of the GoBike system and how the features of users influence their trip duration.

What features in the dataset do you think will help support your investigation into your feature(s) of interest?¶

In order to identify main users, variables such as user type, member year of birth, member gender can be useful. For example, those variables can show whether the system is largely used by local residents or visitors ('user type'), by what age group ('member year of birth'), and by male or female ('member gender'). However, considering the fact that a bike can be used by a various age group, it might be more interesting to investigate user type and gender only. By identifying those features of users, I am planning to investigate if these features influence their trip duration. It would also be useful if there is any data available that can show the travelling distance to explore their travelling pattern, but unfortunately this data is not available on the dataset. However, by looking at the start and end point, we can add a new variable that can show their travelling pattern, such as whether a bike is used for a return journey (i.e. the start point is same as the end point: in this case, perhaps more custmers use the system for touring?) or for short transportation (i.e. the start point is different from the end point: in this case, perhpas more subscribers use the system to commute?). Therefore, we can add this variable to the dataset for further investigation.

Discuss the distribution(s) of your variable(s) of interest. Were there any unusual points? Did you need to perform any transformations?¶

The duration variable took on a large range of values, so I explored the data using a log transform. Under the transformation, the data looked unimodal, with one peak around 600 seconds (10 minutes) with a long tail on the right hand side. Those high outliers were removed and a new data subset was created. Findings from the user type and gender show that a majority of GoBike users are subscribers and male. Moreover, most bikes are not returned to the same start point, but to a different end point.

Of the features you investigated, were there any unusual distributions? Did you perform any operations on the data to tidy, adjust, or change the form of the data? If so, why did you do this?¶

Some of variables contain duplicating information (e.g. station_id, station_name, station_latitude, station_longitude: which all shows a unique location of a station). Instead, I used those location information of the station to create a new variable, called 'return_j', indicating whether a bike returned to the start point or not. Moreover, some variables are stored in a wrong dataformat (e.g. time, and birth year). Therefore, I have removed variables that are not relevant to the data analysis and created a subset. When investigating the gender variable, I found that there is another criteria called 'Other'. There was no extra information available to explain what this 'Other' meant, and therefore, those rows with 'member_gender = other' have been removed, along with 'NaN' rows. Moreover, the variable name 'member_gender' is misleading as it contains the gender information from both subscriber and customer groups.

It explains more about how people might use the GoBike service depending on the user type. For example, from the bivariate explaraton, the majority of users are identified as male and subscribers, and they tend to use the service to travel from one point to another with average ride time of 10 mintues. However, the multivariate exploration shows that customers (targetted for visitors and tourists) tend to keep the bike longer than subscribers (e.g. local residents) although a member pass allows longer ride time (45 minutes) than a casual pass (30 minutes). Moreover, the distribution of duration show that for a bike that returned to the start point, subscribers have two high points for the low duration and high duration, while customers have one high point towards the longer duration. This is perhaps caused by the fact that a subscriber has unlimited 45-minute rides where they could use it as many times as possible as long as it's under 45 mintues, while for customers they paid for a limited use of a bike, and therefore, they tend to use it for a longer time (probably until the maximum limit, which is 30 minutes), rather than a short time.

Were there any interesting or surprising interactions between features?¶

Because the number of male users was significantly higher than the number of female users, I expected that the average duration of male would be higher than female's. However, the multivariate exploration shows the gender difference is most likely to be seen in a different user type (i.e. whether they are customers or subscribers), or in a journey type (i.e. whether a bike was used for the return journey or not).
